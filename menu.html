Basics

To start off, lets get Hello World out of the way:

hello_world.py
print("Hello World!")
Program Output:
>>> Hello World!

Note: ">>>" is used to indicate the program output.

Well, that was easy. Note the lack of semicolons. Lets try a in if statement:

flow_control.py
if (True):
 print("True!")
else:
 print("False!")
Program Output:
>>>True!

No curly braces?
No semi-colons?
Capital "T" True?
What about System.out???

Alright, as Java programmers, there's a lot to digest here. Most importantly is the lack of curly-braces which guides so much of Java's structure. In Python, scope isn't determined by curly-braces, but by tabs. Notice how the places that you'd expect to find braces contain a single colon and then all of the text after it is tabbed up once? It looks completely different but most of the same concepts of scope that apply in Java apply here as well. The colon indicates a new block of code and deeper scope. All of the tabbed code that proceedes is part of that scope!

Not so crazy once it's explained right? Let's get a bit more involved, here's a weird take on the classic Fizz Buzz problem:

fizz_buzz.py
keywords = ["fizz", "buzz", "fizzbuzz"]
divisors = [3, 5]

for i in range(1, 16):
  answer = ""
  for divisor in divisors:
    if (i % divisor == 0):
      answer = answer + keywords[divisors.index(divisor)]
  if (answer == ""):
    print(i)
  else:
    print(answer)
Program Output:
>>> 1
>>> 2
>>> fizz
>>> 4
>>> buzz
>>> fizz
>>> 7
>>> 8
>>> fizz
>>> buzz
>>> 11
>>> fizz
>>> 13
>>> 14
>>> fizzbuzz

As you might start to see, the lack of curly-braces can make it seem difficult to read the code if you're not used to it. This code block begins to show a bit more of Python's syntax. We are introduced to arrays, a for loop, comparisons, and some built-in methods.

We also see that initializing variables in Python couldn't be easier, simply: variable = value no semicolons, and no types!
